# GoArchiver

CLI-архиватор на Go с двумя алгоритмами сжатия: **Huffman Coding** и **Shannon-Fano**.

Проект сделан как pet-проект для портфолио: здесь акцент на архитектуру, работу с бинарными данными и реализацию алгоритмов кодирования без внешних библиотек для компрессии.

## Что умеет

- Упаковывать файл командой `pack`.
- Распаковывать файл командой `unpack`.
- Переключать метод сжатия через флаг `-m` / `--method`:
  - `huffman`
  - `shennon_fano`
- Прогонять тесты и собирать бинарник через `Makefile`.

## Стек

- **Go 1.25.6**
- **Cobra** для CLI
- Стандартная библиотека (`encoding/gob`, `encoding/binary`, `bytes`, `io`, и т.д.)

## Быстрый старт

### 1) Клонирование и запуск тестов/сборки

```bash
make all
```

Команда делает:
- `go test ./...`
- `go build -o moonArchiver ./src`

### 2) Только сборка

```bash
make build
```

После сборки появится бинарник:

```bash
./moonArchiver
```

### 3) Упаковка файла

```bash
./moonArchiver pack path/to/file.txt -m huffman
```

или

```bash
./moonArchiver pack path/to/file.txt -m shennon_fano
```

### 4) Распаковка файла

```bash
./moonArchiver unpack file.txt.vlc -m huffman
```

или

```bash
./moonArchiver unpack file.txt.vlc -m shennon_fano
```

## Как устроен проект

```text
src/
  main.go                          # точка входа
  cmd/                             # CLI-команды (pack/unpack)
  lib/
    compression/                   # общие интерфейсы и кодирование в битовый поток
    chunks/                        # работа с битовыми чанками и байтами
    table/                         # таблицы кодирования + дерево декодирования
      prefix_algoritms/
        huffman/                   # генерация таблицы Хаффмана
        shennon_fano/              # генерация таблицы Shannon-Fano
    compression/algorithms/
      huffman/                     # encode/decode для Хаффмана
      shennon_fano/                # encode/decode для Shannon-Fano
      prefix.go                    # общий формат сериализации prefix-кодов
```

## Архитектурные решения

- Выделены интерфейсы `Encoder` / `Decoder`, чтобы алгоритмы были взаимозаменяемыми.
- Общая логика преобразования в битовый поток вынесена отдельно, алгоритмы переиспользуют инфраструктуру.
- Для Huffman и Shannon-Fano используется общий формат сериализации таблицы кодов (`gob`) вместе с данными, поэтому декодирование автономно.
- В `table` реализовано дерево декодирования, которое подходит для обоих prefix-алгоритмов.

## Алгоритмы

- `huffman`: строит оптимальное prefix-дерево на основе частот символов в конкретном файле.
- `shennon_fano`: строит prefix-коды через рекурсивное разбиение множества символов по частотам.

Оба алгоритма работают с таблицей кодов, которая сохраняется внутри архива вместе с полезными данными.

## Формат выходных файлов

- После упаковки создается файл с суффиксом `.vlc`.
- Если файл с таким именем уже существует, приложение добавляет числовой суффикс, чтобы не перезаписать существующий файл.
- При распаковке исходное имя восстанавливается из имени архива, а при конфликте тоже добавляется суффикс.

Примеры:

```text
report.txt -> report.txt.vlc
report.txt.vlc -> report.txt
report.txt.vlc -> report_1.txt   # если report.txt уже существует
```
